# Changelog

## Unreleased

+ [修复] 注册页状态实时采样号池数量，避免可用账号模式达到目标后继续显示旧的正常账号数。
+ [修复] 注册守护线程增加异常自愈，检查号池、保存状态或代理黑名单清理异常时记录日志并按检查间隔重试，避免后台线程静默停止。
+ [新增] 注册服务状态增加 `runner_alive`；任务仍启用但守护线程停止时，状态读取会自动重启守护线程。
+ [优化] 代理黑名单清理改为按代理刷新间隔自动执行，复用原有手动清理逻辑，不改变黑名单评分和冷却规则。
+ [新增] 注册页代理来源扩展为单代理、代理列表 URL、粘贴代理列表三种模式，URL 和文本模式均支持一行一个代理。
+ [新增] 注册代理池按 worker 线程安全轮询代理；同一个 worker 注册流程内固定使用分配到的代理。
+ [新增] URL 代理列表支持懒刷新和刷新间隔配置；后续刷新失败时保留旧池并展示最后错误，首次拉取失败或为空时明确失败。
+ [新增] 注册页运行状态展示当前代理来源、代理池数量、worker 当前代理和代理池最后错误。
+ [新增] 注册页邮箱配置新增“邮箱服务后台 API 使用注册代理”开关，可让邮箱平台 API 直连，避免被注册代理影响。
+ [修复] 增强 `cloudmail_gen` 当前字段兼容性，支持 `toEmail`、`emailId`、`createTime`、`sendEmail` 等返回字段。
+ [修复] `cloudmail_gen` 对临时网络异常、`429`、`5xx` 增加短重试，降低等待验证码超时概率。
+ [修复] `cloudmail_gen` 在 `emailList` 业务返回异常时自动清理缓存 token、重新 `genToken` 并重试，避免长时间运行后缓存 token 失效导致“邮箱已收到验证码，但页面等待验证码超时”。
+ [新增] 注册代理状态持久化到 `data/register_proxy_state.json`，记录成功次数、失败次数、失败分桶、冷却时间、租约、评分和 platform authorize 平均耗时。
+ [新增] 注册代理池增加 120 秒租约，避免多个并发 worker 在同一时间重复使用同一个代理。
+ [新增] 代理列表 URL / 粘贴代理列表模式启用更激进的免费代理冷却：新代理首次 Cloudflare 拦截冷却 24 小时，新代理硬网络失败冷却 48 小时，历史成功代理采用更温和冷却。
+ [新增] 注册代理选择增加评分排序和新代理探索比例限制；有历史成功代理时优先使用历史成功、低失败、低 platform authorize 耗时的代理。
+ [新增] 注册页增加“重置代理黑名单”按钮，仅在代理列表 URL / 粘贴代理列表模式且存在黑名单记录时显示，确认弹窗会提示当前记录数。
+ [优化] 额度守护模式从暂停恢复注册时重置代理选择周期，重新从代理列表开头选择，同时保留黑名单和评分状态。
+ [优化] 移除代理池全部冷却时的前端打扰提示，后台静默选择可用 fallback，减少长时间挂机时的无意义弹窗。
+ [文档] README 补充本分支相对原版的完整改动、部署方式、上游 PR 链接和代理有效性说明。
+ [文档] 明确本分支部署应使用 `docker-compose.local.yml` 本地构建，避免误用原版官方镜像。
